Review Pull Requests

## Um was geht es?
- Änderungen aus einem Branch in den `main`-Branch übernehmen
- Code vor dem Merge gemeinsam ansehen und besprechen
- Review-Funktionen wie Kommentare und Änderungsvorschläge nutzen
- kompletter Pull-Request-Workflow praktisch durchgeführt

## 5 wichtigste Take-Aways / Befehle / Learnings
- Pull Requests machen Änderungen übersichtlich
- Reviews helfen, Fehler zu finden und Code zu verbessern
- Kommentare direkt an bestimmten Code-Zeilen möglich
- „Suggested changes“ erlauben konkrete Code-Änderungen per Klick
- Merge erst nach Review und Übernahme der Änderungen

## Kurzablauf der Übung (was gemacht wurde)
- Pull Request aus dem Branch `update-game` erstellt
- Pull Request mir selbst als Assignee zugewiesen
- Code unter „Files changed“ überprüft
- Kommentar im Review hinterlassen
- Änderungsvorschlag in `index.html` erstellt
- vorgeschlagene Änderung übernommen (Commit)
- Pull Request in den `main`-Branch gemerged
