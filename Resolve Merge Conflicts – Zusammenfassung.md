# Resolve Merge Conflicts

## Um was geht es?
- was passiert, wenn zwei Leute dieselbe Datei ändern
- warum Git nicht immer automatisch mergen kann
- wie man Merge-Konflikte direkt im Code löst
- Arbeiten mit Pull Requests und Branches
- Konflikte verstehen und keine Angst davor haben

## 5 wichtigste Take-Aways / Learnings
- Merge-Konflikte entstehen, wenn zwei Änderungen sich überschneiden
- Git zeigt genau an, wo der Konflikt ist
- man entscheidet selbst, welche Änderung bleiben soll
- nach dem Lösen muss ein Commit gemacht werden
- mit sauberem Arbeiten gibt es weniger Konflikte

## Kurz was gemacht wurde
- Pull Request erstellt
- Merge-Konflikt im Code geöffnet
- alte und neue Version verglichen
- unnötige Zeilen gelöscht
- Konflikt gespeichert und committet
- Pull Request gemerged
